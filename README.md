## Quick Start

This directory is a self-contained reproduction bundle for the PDF manuscript
*Ordered Stochastic Block Models via Pólya--Gamma data augmentation*.

Run commands from this directory:

```sh
cd "tex file/Ordered Stochastic Block Model"
```

The bundle includes cached MCMC results, post-processing outputs, CSV
summaries, data files, and regenerated plots. This allows the figures and
tables in the PDF to be checked or rebuilt without rerunning the most
expensive MCMC steps.

Most readers will want one of these two routes:

- For the main simulation-study results in the PDF, go to [Simulation study in the main text: Table 1, Table 2, and Figure 4](#simulation-study-in-the-main-text-table-1-table-2-and-figure-4).
- For the main application-study results in the PDF, go to [Application study in the main text: Table 4 and Figures 5-7](#application-study-in-the-main-text-table-4-and-figures-5-7).

## Requirements

Use R with the packages used by the project scripts, in particular:

```r
install.packages(c(
  "ggplot2", "dplyr", "tidyr", "readr", "patchwork", "viridis",
  "igraph", "ggraph", "graphlayouts", "mcclust", "mcclust.ext",
  "salso", "loo", "coda", "truncnorm", "fossil", "Matrix",
  "BayesLogit", "Rcpp", "lpSolve", "scales", "fs"
))
```

Some scripts run without optional packages by using fallbacks, but the list
above matches the environment used to create the bundled results.

## Bundle Contents

- `data/`: empirical network data used by the application analysis.
- `core/`, `helper_folder/`: model fitting and helper functions.
- `scripts/`: renamed top-level entry-point scripts.
- `scripts/analysis/`: analysis drivers, including the one-command simulation post-processing pipeline.
- `output/application/raw/application_run_20260529_110306/`: cached application MCMC fits and CSV summaries.
- `output/posterior_post_processing/application_run_20260529_110306/`: canonical posterior partitions, diagnostics, and K summaries used by plots.
- `output/simulation/raw/`: simulation result CSVs used for the recovery figures and tables.
- `output/simulation/plots/2026-06-15_sst_labels/`: regenerated simulation plots with `SST` labels.
- `output/paper/figures/application_run_20260529_110306_sst_labels/`: regenerated application figures with `SST` labels.
- `paper_figures/all_figures/`: figure assets copied into the bundle for the manuscript PDF.
- `paper_figures/updated plots/`: collected updated application and simulation plots used for the manuscript build.

## Entry Scripts

- `scripts/01_run_application_mcmc.R`: full application MCMC run.
- `scripts/02_run_main_simulation_study.R`: full main simulation study.
- `scripts/03_build_application_postprocessing_cube.R`: builds canonical z-hat/K/diagnostic summaries from application fits.
- `scripts/04_build_paper_tables.R`: rebuilds the application-side tables used in the manuscript PDF.
- `scripts/05_plot_paper_application_figures.R`: rebuilds the application-network figures used in the manuscript PDF.
- `scripts/06_plot_simulation_recovery_figures.R`: rebuilds the main-text and supplementary ARI/VI simulation figures.
- `scripts/07_plot_support_geometry.R`: rebuilds the support-geometry diagnostic figure.
- `scripts/08_plot_mirrored_ocrp_diagnostics.R`: rebuilds mirrored OCRP prior diagnostics.
- `scripts/09_build_bradley_terry_delta_plot.R`: rebuilds only the Bradley--Terry additivity diagnostic.
- `scripts/10_plot_prior_satisfaction_rate.R`: archived diagnostic script for prior-satisfaction checks.
- `scripts/analysis/analyze_simulation.R`: one-command simulation post-processing pipeline for tables and plots.

The copied `scripts/analysis`, `scripts/application`, `scripts/simulation`,
`scripts/diagnostics`, and `scripts/testing` folders are dependencies for these
entry points.

## Full Reruns

Full reruns are computationally expensive and may not reproduce byte-identical
MCMC output unless the same R version, package versions, and random-number
streams are used.

Application MCMC:

```sh
APP_N_ITER=10000 APP_BURN=3000 APP_THIN=2 APP_SEED=42 \
Rscript scripts/01_run_application_mcmc.R
```

Main simulation study:

```sh
Rscript scripts/02_run_main_simulation_study.R
```

After a full simulation rerun, refresh the PDF-facing simulation results with:

```sh
SIM_RESULTS_PATH=output/simulation/raw/<new_simulation_results_csv> \
Rscript scripts/analysis/analyze_simulation.R
```

After a full application rerun, use the new run directory in:

```sh
APP_RUN_DIR=output/application/raw/<new_application_run_id> \
Rscript scripts/03_build_application_postprocessing_cube.R
```

Then rerun:

- [Application study in the main text: Table 4 and Figures 5-7](#application-study-in-the-main-text-table-4-and-figures-5-7)
- [Supplementary application diagnostics in the PDF: Table 9](#supplementary-application-diagnostics-table-9)

with the same `APP_RUN_DIR`.

## Label Update Note

The application and simulation plot scripts in this bundle use `SST` in figure
labels after the geometric section of the manuscript. The old `Toeplitz SST`
labels are not used in the regenerated application plots.
